# 03_get_inlet1_points

This folder generates the Part 1 turbulent-inlet point coordinates for the
SUBOFF workflow. It replaces the dummy `blockMesh` point coordinates with
coordinates derived from the SUBOFF surface grid, uses an OpenFOAM helper mesh to define the rectangular `INLET` patch topology, then exports the inlet patch
coordinates to `INLET1.plt`.

The main downstream output is `INLET1.plt`. Later turbulent-inlet and
boundary-data tools read this file as the inlet point-coordinate list.
`Tecplot_InputFiles/Turb_Inlet.plt` is generated only for visualization and
checking.

## Workflow Idea

The uploaded workflow figure summarizes this folder as the step that generates
the Part 1 turbulent inflow inlet grid coordinates. The `RECT` square region is
treated as an `NL * NL * 3` inlet helper mesh. The coordinates are not generated
directly by `02_grid_use` because the linear interpolation used there gives face
center positions that do not exactly match the OpenFOAM output ordering. This
folder therefore uses OpenFOAM utilities to calculate/export the inlet patch
coordinates.

The OpenFOAM case here is not intended for a solver run. It only needs enough
case structure for mesh utilities:

1. `system/controlDict` must exist.
2. `system/blockMeshDict` defines the rectangular helper mesh and the `INLET`
   patch.
3. `constant/polyMesh_yuanshi/grid.f90` rewrites `constant/polyMesh/points`
   using upstream SUBOFF surface coordinates.
4. `writeMeshObj -constant` exports `patch_INLET_constant.obj`.
5. `cylinder.F90` converts `patch_INLET_constant.obj` into `INLET1.plt` and
   `Tecplot_InputFiles/Turb_Inlet.plt`.

## Repository Layout

```text
03_get_inlet1_points/
├── system/
│   ├── blockMeshDict          # Dummy rectangular OpenFOAM mesh topology
│   └── controlDict            # Minimal OpenFOAM utility case control
├── constant/
│   └── polyMesh_yuanshi/
│       ├── grid.f90           # Rewrites constant/polyMesh/points
│       ├── parameter.h        # Grid dimensions used by grid.f90
│       ├── run.sh             # Builds/runs grid.f90
│       └── ReadData/          # Input files copied from 02_grid_use
├── cylinder.F90               # Converts patch_INLET_constant.obj to .plt files
├── cylinder.h                 # Inlet patch dimensions for cylinder.F90
├── ofrun.sh                   # Runs OpenFOAM writeMeshObj on the constant mesh
├── run.sh                     # Builds/runs cylinder.F90
└── Tecplot_InputFiles/        # Tecplot inspection output
```

## Quick Start

Run each command from the directory named in the step.

1. From `03_get_inlet1_points`, make sure `system/controlDict` exists and check
   the `y/z` mesh resolution in `system/blockMeshDict`.

   Important defaults:

   - `nx 3`: creates four helper point layers in the streamwise direction.
   - `n 160`: creates a `160 * 160` inlet patch.
   - `INLET`: patch exported later by `writeMeshObj`.

2. From `03_get_inlet1_points`, run `blockMesh` to generate
   `constant/polyMesh`.

   ```bash
   blockMesh
   ```

3. Copy the upstream files generated by `02_grid_use` into
   `constant/polyMesh_yuanshi/ReadData/`:

   ```text
   RECT_surface_XYZ.plt
   Y.txt
   ```

4. From `constant/polyMesh_yuanshi`, run the point-coordinate converter:

   ```bash
   ./run.sh
   ```

   This builds and runs `grid.f90`, then writes:

   ```text
   ../polyMesh/points
   ```

   The rewritten `points` file replaces the dummy `blockMesh` coordinates while
   preserving the `faces`, `owner`, `neighbour`, and `boundary` topology already
   created by OpenFOAM.

5. From `03_get_inlet1_points`, export the OpenFOAM mesh patch as OBJ:

   ```bash
   sbatch ofrun.sh
   ```

   `ofrun.sh` sources the OpenFOAM v2312 environment and runs:

   ```bash
   writeMeshObj -constant
   ```

   The expected intermediate file is:

   ```text
   patch_INLET_constant.obj
   ```

6. From `03_get_inlet1_points`, convert the exported OBJ patch to inlet
   coordinate files:

   ```bash
   ./run.sh
   ```

   This builds and runs `cylinder.F90`, then writes:

   ```text
   INLET1.plt
   Tecplot_InputFiles/Turb_Inlet.plt
   ```

## Input Files

| File | Used By | Meaning |
| --- | --- | --- |
| `constant/polyMesh_yuanshi/ReadData/RECT_surface_XYZ.plt` | `grid.f90` | Square H-type surface coordinates generated by `02_grid_use`. |
| `constant/polyMesh_yuanshi/ReadData/Y.txt` | `grid.f90` | Wall-normal distance list generated by `02_grid_use`. |
| `patch_INLET_constant.obj` | `cylinder.F90` | OpenFOAM utility output for the `INLET` patch after `points` is rewritten. |

## Output Files

| File | Meaning |
| --- | --- |
| `constant/polyMesh/points` | OpenFOAM `vectorField` written by `grid.f90`; consumed by OpenFOAM mesh utilities. |
| `patch_INLET_constant.obj` | Temporary OBJ output from `writeMeshObj -constant`. |
| `INLET1.plt` | Raw `NALL` inlet coordinates for downstream turbulent-inlet workflows. |
| `Tecplot_InputFiles/Turb_Inlet.plt` | Tecplot visualization file for checking the turbulent inlet patch grid. |

## Important Parameters

The grid dimensions must stay consistent across `blockMeshDict`,
`parameter.h`, and `cylinder.h`.

| Location | Parameter | Meaning |
| --- | --- | --- |
| `constant/polyMesh_yuanshi/parameter.h` | `NK=640` | Circumferential resolution from the upstream grid. |
| `constant/polyMesh_yuanshi/parameter.h` | `NL=NK/4` | Inlet square-side resolution; default `NL=160`. |
| `constant/polyMesh_yuanshi/parameter.h` | `NI=400` | Wall-normal index range read from `Y.txt`. |
| `system/blockMeshDict` | `n 160` | Inlet patch face count in each `y/z` direction; must match `NL`. |
| `system/blockMeshDict` | `nx 3` | Creates four helper point layers, matching the `grid.f90` loop from `NI` to `NI-3`. |
| `cylinder.h` | `N=160` | Inlet patch resolution used when reading `patch_INLET_constant.obj`. |
| `cylinder.h` | `NALL=N*N` | Total number of inlet patch points written to `INLET1.plt`. |

## Ordering Notes

`grid.f90` writes OpenFOAM points in this order:

```fortran
DO J=NL+1,1,-1
DO K=1,NL+1
    DO I=NI,NI-3,-1
```

This ordering must match the OpenFOAM connectivity generated by `blockMesh`.
Changing the loop order, `blockMeshDict` dimensions, or `NL` without updating
the related files can break the patch coordinate ordering.

`cylinder.F90` writes `Tecplot_InputFiles/Turb_Inlet.plt` with `I=N`, `J=N`.
The source comment states that the visualization ordering is `z` direction
first, with `z` from `ZMIN` to `ZMAX` and `y` from `YMAX` to `YMIN`.

## Notes And Limitations

- This folder is a mesh and coordinate conversion helper, not a complete solver
  case. The absence of `0` fields and full physical `constant` dictionaries is
  intentional.
- `ofrun.sh` contains cluster-specific SLURM and OpenFOAM environment settings.
  Update the partition, module setup, or OpenFOAM source path for a different
  machine.
- Both `run.sh` scripts call `make`. If `make` fails, restore or provide the
  expected local build rules for the Fortran programs.
- If `NK`, `NL`, `n`, or `N` changes, regenerate the OpenFOAM mesh and keep all
  dimension definitions synchronized.
