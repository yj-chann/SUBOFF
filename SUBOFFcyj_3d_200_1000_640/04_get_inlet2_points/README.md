# 04_get_inlet2_points

This folder generates the Part 2 laminar-inlet point coordinates for the
SUBOFF workflow. It handles the O-grid part of the inlet: OpenFOAM first
creates a rectangular helper mesh and patch topology, `grid.f90` replaces the
dummy `points` coordinates with O-grid coordinates derived from the upstream
SUBOFF surface, and `cylinder.F90` exports the final `INLET` patch coordinates.

The main downstream output is `INLET2.plt`. `Tecplot_InputFiles/Lam_Inlet.plt`
is generated for visualization and checking of the laminar-inlet grid.

## Workflow Idea

It generates the laminar inlet points for the O-type grid
region. The folder uses OpenFOAM mesh utilities to preserve the patch ordering
expected by downstream OpenFOAM and inflow-generation tools.

The OpenFOAM case here is a mesh/coordinate conversion helper, not a complete
solver case. It only needs enough case structure for mesh utilities:

1. `system/controlDict` must exist.
2. `system/blockMeshDict` creates `constant/polyMesh` and defines the `INLET`
   patch.
3. `constant/polyMesh_yuanshi/grid.f90` rewrites `constant/polyMesh/points`
   using upstream O-grid surface coordinates.
4. `writeMeshObj -constant` exports `patch_INLET_constant.obj`.
5. `cylinder.F90` converts `patch_INLET_constant.obj` into `INLET2.plt` and
   `Tecplot_InputFiles/Lam_Inlet.plt`.

## Repository Layout

```text
04_get_inlet2_points/
|-- system/
|   |-- blockMeshDict          # Dummy rectangular OpenFOAM mesh topology
|   `-- controlDict            # Minimal OpenFOAM utility case control
|-- constant/
|   `-- polyMesh_yuanshi/
|       |-- grid.f90           # Rewrites constant/polyMesh/points
|       |-- parameter.h        # Grid dimensions used by grid.f90
|       |-- run.sh             # Builds/runs grid.f90
|       `-- ReadData/          # Input files copied from 02_grid_use
|-- cylinder.F90               # Converts patch_INLET_constant.obj to .plt files
|-- cylinder.h                 # Inlet patch dimensions for cylinder.F90
|-- ofrun.sh                   # Runs OpenFOAM writeMeshObj on the constant mesh
|-- run.sh                     # Builds/runs cylinder.F90
`-- Tecplot_InputFiles/        # Tecplot inspection output
```

## Quick Start

Run each command from the directory named in the step.

1. From `04_get_inlet2_points`, make sure `system/controlDict` exists and check
   the `y/z` mesh resolution in `system/blockMeshDict`.

   Important defaults:

   - `nx 3`: creates four helper point layers in the streamwise direction.
   - `ny 920`: matches `NJ=NJ1+NJ2+NJ3`.
   - `nz 640`: matches `NK`.
   - `INLET`: patch exported later by `writeMeshObj`.

2. From `04_get_inlet2_points`, run `blockMesh` to generate
   `constant/polyMesh`.

   ```bash
   blockMesh
   ```

3. Copy the upstream files generated by `02_grid_use` into
   `constant/polyMesh_yuanshi/ReadData/`:

   ```text
   CY_surface_XYZ.plt
   Y.txt
   ```

4. From `constant/polyMesh_yuanshi`, run the point-coordinate converter:

   ```bash
   ./run.sh
   ```

   This builds and runs `grid.f90`, then writes:

   ```text
   ../polyMesh/points
   ```

   The rewritten `points` file replaces the dummy `blockMesh` coordinates while
   preserving the `faces`, `owner`, `neighbour`, and `boundary` topology already
   created by OpenFOAM.

5. From `04_get_inlet2_points`, export the OpenFOAM mesh patch as OBJ:

   ```bash
   sbatch ofrun.sh
   ```

   `ofrun.sh` sources the OpenFOAM v2312 environment and runs:

   ```bash
   writeMeshObj -constant
   ```

   The expected intermediate file is:

   ```text
   patch_INLET_constant.obj
   ```

6. From `04_get_inlet2_points`, convert the exported OBJ patch to inlet
   coordinate files:

   ```bash
   ./run.sh
   ```

   This builds and runs `cylinder.F90`, then writes:

   ```text
   INLET2.plt
   Tecplot_InputFiles/Lam_Inlet.plt
   ```

## Input Files

| File | Used By | Meaning |
| --- | --- | --- |
| `constant/polyMesh_yuanshi/ReadData/CY_surface_XYZ.plt` | `grid.f90` | Combined cylindrical/O-type surface coordinates generated by `02_grid_use`. |
| `constant/polyMesh_yuanshi/ReadData/Y.txt` | `grid.f90` | Wall-normal distance list generated by `02_grid_use`; `grid.f90` uses `Y(NI)` for the laminar inlet offset. |
| `patch_INLET_constant.obj` | `cylinder.F90` | OpenFOAM utility output for the `INLET` patch after `points` is rewritten. |

## Output Files

| File | Meaning |
| --- | --- |
| `constant/polyMesh/points` | OpenFOAM `vectorField` written by `grid.f90`; consumed by OpenFOAM mesh utilities. |
| `patch_INLET_constant.obj` | Temporary OBJ output from `writeMeshObj -constant`. |
| `INLET2.plt` | Raw `NALL` laminar-inlet coordinates for downstream workflows. |
| `Tecplot_InputFiles/Lam_Inlet.plt` | Tecplot visualization file for checking the laminar O-grid inlet. |

## Important Parameters

The grid dimensions must stay consistent across `blockMeshDict`,
`parameter.h`, and `cylinder.h`.

| Location | Parameter | Meaning |
| --- | --- | --- |
| `constant/polyMesh_yuanshi/parameter.h` | `NJ1=150`, `NJ2=450`, `NJ3=320` | Streamwise O-grid sections from the upstream mesh. |
| `constant/polyMesh_yuanshi/parameter.h` | `NJ=NJ1+NJ2+NJ3` | Total streamwise inlet count; default `NJ=920`. |
| `constant/polyMesh_yuanshi/parameter.h` | `NK=640` | Circumferential O-grid resolution. |
| `constant/polyMesh_yuanshi/parameter.h` | `NI=200` | Wall-normal index range read from `Y.txt`. |
| `system/blockMeshDict` | `ny 920` | Inlet patch streamwise face count; must match `NJ`. |
| `system/blockMeshDict` | `nz 640` | Inlet patch circumferential face count; must match `NK`. |
| `system/blockMeshDict` | `nx 3` | Creates four helper point layers, matching the `grid.f90` loop `I=0,3`. |
| `cylinder.h` | `NALL=NK*NJ` | Total number of inlet patch points written to `INLET2.plt`. |

## Ordering Notes

`grid.f90` reads `CY_surface_XYZ.plt` in this order:

```fortran
DO K=1,NK
DO J=1,NJ+1
    READ(20,*) CY_X0(J,K),CY_Y0(J,K),CY_Z0(J,K)
```

It writes OpenFOAM points in this order:

```fortran
DO K=1,NK
DO J=1,NJ+1
DO I=0,3
    WRITE(2,230) '( ',XP,' ',YP,' ',ZP,' )'
```

After that, it writes an additional `K=1` circumferential copy so the O-grid
topology is represented as a single connected OpenFOAM block:

```fortran
DO K=1,1
DO J=1,NJ+1
DO I=0,3
```

`cylinder.F90` reads `NALL=NK*NJ` vertices from `patch_INLET_constant.obj`.
For Tecplot visualization, `Lam_Inlet.plt` is written as `I=NJ`, `J=NK+1`;
the extra `NJ` points close the circumferential plotting seam. The raw
`INLET2.plt` file does not include that duplicate Tecplot seam.

The source comment states that visualization ordering is streamwise direction
first, from `SMIN` to `SMAX`, with circumferential direction counter-clockwise
from `-45` degrees to `315` degrees.

## Notes And Limitations

- This folder is a mesh and coordinate conversion helper, not a complete solver
  case. The absence of `0` fields and full physical `constant` dictionaries is
  intentional.
- `ofrun.sh` contains cluster-specific SLURM and OpenFOAM environment settings.
  Update the partition, job name, module setup, or OpenFOAM source path for a
  different machine.
- Both `run.sh` scripts call `make`. If `make` fails, restore or provide the
  expected local build rules for the Fortran programs.
- If `NJ1`, `NJ2`, `NJ3`, `NJ`, `NK`, `ny`, `nz`, or `NALL` changes, regenerate
  the OpenFOAM mesh and keep all dimension definitions synchronized.
