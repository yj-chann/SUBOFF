# 05_ini_2d_RAS

This folder runs the 2D RAS initialization case used to build non-uniform inlet
pressure and velocity fields for the later 3D SUBOFF cases. It uses the
extended 2D meridional grid from `02_grid_use`, solves a steady
`simpleFoam + RAS` problem, then interpolates the converged 2D pressure and
velocity onto the Part 1 square inlet (`INLET1`) and Part 2 O-grid inlet
(`INLET2`).

The main downstream outputs are `get_ini_p_u/PU_RECT_SUR.plt` and
`get_ini_p_u/PU_CY_SUR.plt`. Tecplot files under
`get_ini_p_u/Tecplot_InputFiles` are for visualization and checking.

## Workflow Idea

This step is the 2D precursor calculation used
to reduce the artificial uniform-inlet limitation of the smaller 3D domain.
Instead of imposing a constant inlet profile directly in the 3D calculation,
the larger 2D extended domain is solved first. Its resulting pressure and
velocity distribution is sampled by streamwise `x` location and mapped onto the
3D inlet point sets from:

- `03_get_inlet1_points/INLET1.plt` for the rectangular inlet part.
- `04_get_inlet2_points/INLET2.plt` for the cylindrical O-grid inlet part.

`get_ini_p_u/cylinder.F90` performs the interpolation by `x` coordinate. The 2D
velocity components are treated as streamwise and radial components: `Ux` is
kept as the streamwise velocity, while the radial velocity is projected onto
the 3D inlet coordinates as `Uy` and `Uz`.

## Quick Start

Run each command from the directory named in the step.

1. From `05_ini_2d_RAS`, check that `system/controlDict` exists and that
   `system/blockMeshDict` matches the extended 2D grid dimensions from
   `suboff_mesh_2d_ADD.plt`.

   Important defaults:

   - `simpleFoam` is the solver.
   - `simulationType RAS` with `kOmegaSST` is used.
   - `blockMeshDict` uses `(440 981 1)`, matching `M1=441` and `M2=982`.
   - `FRONT` and `BACK` are wedge patches; `AXIS` is empty.

2. From `05_ini_2d_RAS`, run `blockMesh` to generate `constant/polyMesh`.

   ```bash
   blockMesh
   ```

3. Copy `02_grid_use/suboff_mesh_2d_ADD.plt` into:

   ```text
   constant/polyMesh_yuanshi/ReadData/suboff_mesh_2d_ADD.plt
   ```

4. From `constant/polyMesh_yuanshi`, rewrite the OpenFOAM `points` file:

   ```bash
   ./run.sh
   ```

   This builds and runs `cylinder.F90`, then writes:

   ```text
   ../polyMesh/points
   ```

   The rewritten `points` file preserves the OpenFOAM topology generated by
   `blockMesh` while replacing the dummy coordinates with the extended 2D SUBOFF
   grid.

5. Run the OpenFOAM RAS calculation from `05_ini_2d_RAS`.

   The reference workflow uses a cluster submission command such as:

   ```bash
   sbatch ofrun
   ```

   In this folder snapshot, no root-level OpenFOAM run script is present, so use
   the local project run script for this case or run the usual OpenFOAM sequence
   for the available decomposition settings:

   ```bash
   decomposePar
   mpirun -np 64 simpleFoam -parallel
   reconstructPar
   ```

   `get_ini_p_u/cylinder.F90` currently reads the final fields from:

   ```text
   ../50000/p
   ../50000/U
   ```

6. Prepare interpolation inputs under `get_ini_p_u/ReadData/`:

   ```text
   suboff_mesh_2d_ADD.plt
   INLET1.plt
   INLET2.plt
   ```

   `INLET1.plt` comes from `03_get_inlet1_points`, and `INLET2.plt` comes from
   `04_get_inlet2_points`.

7. From `get_ini_p_u`, run the interpolation program:

   ```bash
   ./run.sh
   ```

   This builds and runs `cylinder.F90`, then writes the inlet pressure/velocity
   files listed below.

## Important Parameters

The grid dimensions must stay consistent across `blockMeshDict`, both
`cylinder.h` files, `02_grid_use`, and the inlet point files.

| Location | Parameter | Meaning |
| --- | --- | --- |
| `system/blockMeshDict` | `(440 981 1)` | Cell dimensions for the 2D wedge helper mesh. |
| `constant/polyMesh_yuanshi/cylinder.h` | `M1=400+1+40` | Wall-normal point count from `suboff_mesh_2d_ADD.plt`; default `441`. |
| `constant/polyMesh_yuanshi/cylinder.h` | `M2=980+1+1` | Streamwise point count from `suboff_mesh_2d_ADD.plt`; default `982`. |
| `constant/polyMesh_yuanshi/cylinder.h` | `M3=640` | Circumferential count used to set the wedge angles. |
| `get_ini_p_u/cylinder.h` | `NI=400` | Original OpenFOAM wall-normal index count. |
| `get_ini_p_u/cylinder.h` | `NL=NK/4` | Square inlet side resolution; default `160`. |
| `get_ini_p_u/cylinder.h` | `NJ=NJ1+NJ2+NJ3` | O-grid streamwise count; default `900`. |
| `get_ini_p_u/cylinder.h` | `NK=640` | O-grid circumferential count. |
| `system/decomposeParDict` | `numberOfSubdomains 64` | Parallel decomposition count for the OpenFOAM solve. |

## Interpolation And Ordering Notes

The 2D solution is read from OpenFOAM field files after skipping the 23-line
header. The code assumes the field values are ordered with the wall-normal index
inside the streamwise index:

```fortran
DO J=1,N2
DO I=1,N1
    READ(1,*) PREF(I,J)
```

The sampled reference line uses:

```fortran
I=N1-NI+1
```

For each inlet point, `get_ini_p_u/cylinder.F90` searches for the bracketing
`x` interval in the sampled 2D line and linearly interpolates `p`, `Ux`, and the
radial velocity. The radial component is projected into 3D as:

```fortran
Uy = Ur * Y / sqrt(Y**2 + Z**2)
Uz = Ur * Z / sqrt(Y**2 + Z**2)
```

This projection is why `PU_RECT_SUR.plt` and `PU_CY_SUR.plt` contain seven
columns:

```text
X Y Z p Ux Uy Uz
```

## Time Step Check

The reference notes mention estimating the time step from maximum speed and
minimum grid spacing:

```text
DT ~ dx_min / Umag_max
```

This README does not prescribe a single value because it depends on the actual
generated grid and the final velocity field. Recheck `Umag_max` and `dx_min`
after changing `suboff_mesh_2d_ADD.plt`, inlet velocity, or mesh dimensions.

## Notes And Limitations

- This folder produces initialization data for later 3D cases; it is not the
  final 3D simulation.
- The path `../50000/` is hard-coded in `get_ini_p_u/cylinder.F90`. If the RAS
  case ends at another time directory, update the code or copy the desired
  fields to `50000`.
- If mesh dimensions change, update `system/blockMeshDict`,
  `constant/polyMesh_yuanshi/cylinder.h`, `get_ini_p_u/cylinder.h`, and the
  upstream `02_grid_use` outputs together.
- Check Tecplot outputs before copying `PU_RECT_SUR.plt` and `PU_CY_SUR.plt`
  into downstream `ReadData` folders.
