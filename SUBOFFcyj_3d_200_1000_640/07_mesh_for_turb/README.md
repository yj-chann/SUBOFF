# 07_mesh_for_turb

This folder builds a 3-layer OpenFOAM domain for applying the turbulent
inlet generated by `06_tg` and calculating the mass flow. It shifts the `INLET1` square inlet grid inward to form a thin block, creates OpenFOAM `constant/boundaryData/INLET` time-varying
velocity input, and runs `pimpleFoam` with `timeVaryingMappedFixedValue` on the
`INLET` patch.

The case is mainly a validation and preparation step: it checks that the
synthetic turbulent inlet files can be mapped into OpenFOAM and provides probe
and mass-flow data for the first inlet-cell layer.

## Workflow Idea

The mesh is a thin rectangular block made from the square `INLET1` face grid.
The inlet and hull-side patches are OpenFOAM patches, while the other side
faces are symmetry patches. The inlet boundary condition reads OpenFOAM
`boundaryData`, so the `INLET` directory must contain:

- `points`: the `INLET1` point coordinates.
- time-named subdirectories such as `0`, `0.0002`, `0.0004`, ...
- one `U` file inside each time directory.

The velocity values in those `U` files are converted from the `06_tg/vel`
outputs. The time spacing must match the turbulent-inlet generator time step
`DT`.

## Quick Start

Run each command from the directory named in the step.

1. Generate the probe include files.

   Copy these inputs into `system/hull_probes_out_of/ReadData/`:

   ```text
   INLET1.plt
   Y.txt
   ```

   `INLET1.plt` comes from `03_get_inlet1_points`; `Y.txt` comes from
   `02_grid_use`. Then run:

   ```bash
   cd system/hull_probes_out_of
   ./run.sh
   ```

   This writes directly into `system/`:

   ```text
   probesInletShift.c
   probesInletShiftCenterLine.c
   ```

   `controlDict` includes these files in `probes1` and `probes2`.

2. Generate the base OpenFOAM topology.

   From `07_mesh_for_turb`, check `system/blockMeshDict`, especially:

   - `nx 3`: three cells normal to the inlet plane.
   - `n 160`: square inlet resolution matching `INLET1`.
   - `OUTLET`: symmetry side patch.
   - `HULL`: patch used as the downstream side of the thin block.

   Then run:

   ```bash
   blockMesh
   ```

3. Replace the dummy blockMesh point coordinates.

   Copy upstream files into `constant/polyMesh_yuanshi/ReadData/`:

   ```text
   RECT_surface_XYZ.plt
   Y.txt
   ```

   Then run:

   ```bash
   cd constant/polyMesh_yuanshi
   ./run.sh
   ```

   `grid.f90` writes:

   ```text
   ../polyMesh/points
   ```

   The translation distance for the 3 inner layers is:

   ```text
   Y(NI) - Y(NI-1)
   ```

4. Generate manual decomposition if using manual decomposition.

   ```bash
   cd constant/manualDecomposition_code
   ./run.sh
   ```

   This writes:

   ```text
   constant/manualDecomposition
   ```

   The current `system/decomposeParDict` uses `method scotch`, so this manual
   file is optional unless `method manual` is enabled.

5. Create the `constant/boundaryData/INLET` time directories.

   ```bash
   cd constant/boundaryData/mkdir_code
   ./run.sh
   ```

   `mkdir.F90` creates a local `ofrun.sh` containing `mkdir -p` commands and
   then executes it. The directory names cover `0` through `DT*NT`.

6. Generate the mapped inlet `points` file.

   Copy `03_get_inlet1_points/INLET1.plt` into:

   ```text
   constant/boundaryData/points_code/ReadData/INLET1.plt
   ```

   Then run:

   ```bash
   cd constant/boundaryData/points_code
   ./run.sh
   ```

   This writes:

   ```text
   constant/boundaryData/INLET/points
   ```

7. Convert `06_tg` velocity files into OpenFOAM boundaryData.

   From `constant/boundaryData/inlet_code`, run:

   ```bash
   ./run.sh
   ```

   `suboff_main.F90` reads:

   ```text
   ../../../../06_tg/vel/u000001.plt
   ../../../../06_tg/vel/u000002.plt
   ...
   ```

   and writes OpenFOAM vector lists such as:

   ```text
   constant/boundaryData/INLET/0/U
   constant/boundaryData/INLET/0.0002/U
   constant/boundaryData/INLET/0.0004/U
   ```

   The first velocity file, `u000001.plt`, is used for both `0/U` and the first
   nonzero time step.

8. Decompose and run the OpenFOAM case.

   The reference workflow mentions submitting one job for `decomposePar` and
   another for `pimpleFoam`. In this folder snapshot there is no root-level
   `ofrun.sh`, so use the local project job script if present in your working
   copy, or run the commands directly:

   ```bash
   decomposePar
   mpirun -np 64 pimpleFoam -parallel
   reconstructPar
   ```

## BoundaryData Notes

The folder name under `constant/boundaryData` must match the OpenFOAM patch name
exactly. Here the patch is `INLET`, so the directory must be:

```text
constant/boundaryData/INLET
```

For `timeVaryingMappedFixedValue`, OpenFOAM expects:

```text
constant/boundaryData/INLET/points
constant/boundaryData/INLET/0/U
constant/boundaryData/INLET/0.0002/U
constant/boundaryData/INLET/0.0004/U
...
```

Do not rename `boundaryData` or `INLET`. If the patch name changes in
`blockMeshDict`, update `0/U`, `constant/boundaryData/<patchName>`, and every
helper script that writes `../INLET/...`.

## Ordering Notes

`constant/polyMesh_yuanshi/grid.f90` writes the mesh points in this order:

```fortran
DO J=NL+1,1,-1
DO K=1,NL+1
    WRITE outer point
    DO I=1,3
        WRITE translated inner point
```

`constant/boundaryData/points_code/cylinder.F90` writes `INLET/points` in the
same `INLET1.plt` order:

```fortran
DO I=1,NALL
    WRITE INLET_X(I), INLET_Y(I), INLET_Z(I)
```

The time-varying `U` files must use the same point ordering as
`INLET/points`. Otherwise `timeVaryingMappedFixedValue` will map velocities to
the wrong inlet locations.

## Notes And Limitations

- This case validates and maps the Part 1 turbulent inlet only. It does not
  include the O-grid `INLET2` region.
- The current repository snapshot has shell scripts that call `make`, but no
  `Makefile` files were found under this folder. Restore or provide the local
  build rules before running the helper programs.
- `constant/boundaryData/inlet_code/suboff_main.F90` reads `06_tg/vel` using a
  fixed relative path. Keep the folder layout unchanged or update `DIR`.
- To avoid input/output file collisions in the MPI boundaryData converter, file
  unit numbers are tied to `iMPI_MyID`. Keep `NPP`, `NT`, and `ST` consistent
  when changing the parallel layout.
- Keep `DT` synchronized across `06_tg/parameter.h`, `system/controlDict`,
  `mkdir_code/parameter.h`, and `inlet_code/parameter.h`.
- Probe files were originally for validating the probe workflow. They can be
  disabled later by removing or commenting the `probes1` and `probes2` function
  objects in `system/controlDict`.
