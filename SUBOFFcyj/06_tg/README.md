# 06_tg

This folder generates a time series of synthetic turbulent velocity inlets for
the square Part 1 inlet grid. The current implementation in `tg_parallel.F90`
uses a von Karman-Pao energy spectrum to build an isotropic, divergence-free
fluctuating velocity field, convects that field past `INLET1` with Taylor's
frozen-turbulence assumption, removes the sampled time mean, and writes one
velocity file per time step under `vel/`.

The main downstream outputs are:

- `vel/u000001.plt` through `vel/uNTALL.plt`: one square-inlet velocity field
  per time step, containing `u v w` at each `INLET1` point.
- `Tecplot_InputFiles/UModify_INLET.plt`: first generated inlet field with
  coordinates for inspection.
- `Tecplot_InputFiles/UTIME_CenterLine_INLET.plt`: center-point velocity time
  series for checking the generated fluctuations.

## Workflow Idea

The turbulent inlet varies with time. For each time step, this program samples a
3D synthetic isotropic turbulence block at the moving inlet-plane positions and
writes the resulting inlet velocity field to `vel/`. The code is MPI-parallel:
each rank handles `NTP=NTALL/NPP` consecutive time steps, and the combined
output covers time steps `1...NTALL`.

The current code generates a uniform-mean turbulent inlet:


$U_{inlet}(x, t) = U_{REF} e_x + u'(x - U_{REF} t)$


The first pass samples the synthetic turbulence and computes its time mean on
the inlet grid. The second pass subtracts that sampled mean and adds the uniform
streamwise velocity `UREF`. This makes the generated fluctuating component
approximately zero-mean over the `NTALL` sampled time steps.

The notes for the wider workflow also mention superposing the fluctuations on
`PU_RECT_SUR.plt` from `05_ini_2d_RAS` to create a non-uniform inlet. In this
folder snapshot, `tg_parallel.F90` does not read `PU_RECT_SUR.plt`; it adds the
uniform mean `UREF` in the second pass.

## Method: Isotropic Uniform Turbulent Inlet

The generator follows the synthetic turbulence idea described in the reference
notes: represent a homogeneous isotropic velocity fluctuation as a finite sum
of random Fourier modes.

For position vector `x`, the fluctuating velocity is assembled as:


$u'(x) = \sum\limits_{m=1}^M 2 \sqrt{E(k_m) \Delta k} \cos(k_m \hat{\mathbf{k}}_m . \mathbf{x} - \psi_m) \hat{\sigma}_m$


where:

- `k_m` is a random wave vector with magnitude between `k_min` and `k_max`.
- `psi_m` is a random phase.
- `E(k_m)` is the von Karman-Pao energy spectrum.
- `sigma_m` is a unit velocity direction perpendicular to the corrected wave
  vector, so the generated field is approximately divergence-free.

### Wave Numbers

The minimum wave number is set by the largest resolved domain scale:

```fortran
km0 = max(2.0*M_PI/lx, 2.0*M_PI/ly, 2.0*M_PI/lz)
```

The maximum wave number is set by the grid resolution:

```fortran
kmmax = M_PI/dx
```

The wave-number spacing is:

```fortran
dk = (kmmax-km0)/nModes
```

Random angles `theta` and `phi` define each wave-vector direction:

```fortran
kx = sin(theta)*cos(phi)*km
ky = sin(theta)*sin(phi)*km
kz = cos(theta)*km
```

### Divergence-Free Direction

The code uses a discrete corrected wave vector:

```fortran
ktx = sin(kx*dx/2)/dx
kty = sin(ky*dy/2)/dy
ktz = sin(kz*dz/2)/dz
```

Then it takes a cross product between a random unit vector `zeta` and this
corrected wave vector:

```text
sigma_m = normalize(zeta_m x k_tilde_m)
```

This makes each mode perpendicular to `k_tilde_m`, which suppresses artificial
compressibility in the synthetic velocity field.

### von Karman-Pao Spectrum

The energy spectrum is computed in `subroutine karman`:

```text
E(k) = alpha * (urms^2 / ke)
       * ((k/ke)^4 / (1 + (k/ke)^2)^(17/6))
       * exp(-2 (k/keta)^2)
```

with the current constants:

```fortran
Lin  = 0.005
nu   = 1.31e-6
urms = UREF*0.6
ke   = 0.746834/Lin
alpha = 1.452762113
epsilone = urms**3/Lin
keta = epsilone**0.25 * nu**(-3.0/4.0)
```

`Tecplot_InputFiles/von Karman-Pao.plt` is written by rank 0 for checking the
resulting spectrum.

### Time Series Sampling

The turbulence block is treated as frozen and convected at `UREF`. For each
time step and inlet point, the sampled position is:

```fortran
INLET_XP = INLET_X + XREF0 - time*UREF
INLET_YP = INLET_Y + YREF0
INLET_ZP = INLET_Z + ZREF0
```

The synthetic velocity field is trilinearly interpolated from the turbulence
block to the moving inlet point. After the first pass computes the inlet-grid
time mean, the second pass writes:

$$U_{new} = U_{sample} + UREF e_x - mean(U_{sample})$$


to `vel/u######.plt`.

## Coverage Constraint

Each MPI rank stores only `nxp` planes of the synthetic turbulence block. The
local `x` window must be wide enough to cover every moving inlet point handled
by that rank during its `NTP` time steps.

The practical constraint from the workflow notes is:

```text
nxp * dx > DT * NTP * UREF + (max(INLET_X) - min(INLET_X))
```

The code checks the sampled `x` index during interpolation and aborts if a
point falls outside the rank's local window. If this happens, increase `nxp`,
reduce `NTP`, reduce `DT`, or enlarge the synthetic turbulence block.

## Notes And Limitations

- `ReadData/INLET1.plt` must contain exactly `NXALL=N1*N1` points in the same
  ordering expected by `03_get_inlet1_points`.
- `NTALL` must be divisible by `NPP`, because `NTP=NTALL/NPP` is used directly.
- `ofrun.sh` contains cluster-specific module and path settings. Update the
  `cd` path, partition, module, and MPI command for a different machine.
- `Makefile` is tied to `mpiifort`, Intel MKL include paths, and a local
  `fftw2xf` library path. Update those paths before building elsewhere.
- Only rank 0 creates the random phases and broadcasts them, so all ranks sample
  one consistent turbulence field.
- The current code generates a uniform-mean turbulent inlet. It is designed to temporarily      neglect `PU_RECT_SUR.plt`. The implmentation of `PU_RECT_SUR.plt` is in `07_mesh_for_turb`.
